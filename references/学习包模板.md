# 学习包交付模板（4 份）

落盘到 `~/Downloads/学习包/YYYYMMDD-[主题]/`：**00 是一页思维导图 PNG 图片**（HTML 排版 + scripts/html2png.sh 截图），**01/02/03 是 Markdown**（转 PDF 时由 scripts/md2pdf.sh 套电子书排版：A4 / 680px / 棕褐主色 / 衬线标题）。所有叙事中文，英文原文双语，命令/代码保留原文 + 中文解释。叙事正文避免破折号式写法；URL、代码、命令、Markdown 表格语法、文件名、英文专有名词保留原样。

**学习包难度必须在 00/01/02/03 都标明并执行**：入门级 = 普通人友好，少术语、先讲直觉和场景、只放最少够用资源；中级 = 核心术语 + 方法框架 + 代表案例；进阶级 = 可加入官方深文档、论文、源码、复杂案例和行业讨论。用户没选时默认入门级。

**四份分工（互不重叠）**：00 管「怎么走」（路线/顺序/验证，不展开概念、不铺 URL）｜01 管「用什么料」（资料清单 + 术语一行速查，不讲透概念）｜02 管视频｜03 管「把知识讲透」（电子书式正文，不列资源库、不做路线）。概念深入只在 03，术语速查只在 01，资源 URL 只在 01，阶段节奏只在 00。

## 03-学习笔记.md（本 skill 直接产出的 Markdown 笔记，转 PDF 套电子书排版）

定位：电子书风格的**学习笔记正文**，内容优先、无插画。把主题核心知识用人话讲透，能脱离原始链接独立读懂。**不调下游 skill、不生成图片。** 文件是纯 Markdown，电子书版式在 Stage 5 转 PDF 时由 `scripts/md2pdf.sh` 套上（A4 / 680px / 棕褐主色 / 衬线标题）。

写作要求：
- **书名**：一级标题 `# 《[主题]学习笔记》`，三级标题写副标题，引用块写一行 meta（难度/程度/基础/日期 + 指路 00/01/02）。
- **章节骨架**：前言 → 第 1 章 [核心知识块] → … → 结语。
- **难度执行**：入门级先讲生活场景和直觉，再给术语，少用缩写；中级可以直接使用核心术语，但每章仍要配例子；进阶级可以讨论复杂机制、源码/论文/行业分歧。
- **详细度按 Stage 1 选择**（默认适中加厚）：适中加厚 = 每章 2 到 3 个概念小节（`###`各带例子）+ 一个场景示范 + 常见坑 1 到 2 条 + 一句小结；详细教程版 = 再加可运行代码走读、常见报错排查、章末小练习，篇幅 1.5 到 2 倍。
- **每章内容**：讲解口吻把概念讲明白，每个概念小节至少一个例子/打比方；关键处引权威原话（英文原句 + 中文翻译 + 真实 URL）；代码用 ``` 围栏 + 中文注释。
- **竖条铁律**：提示框/引用块只在「引权威原话」时用 `>`，每章最多一处；例子用正文 + 有序列表，**不要把例子、要点、引用都套成块**（否则 PDF 里排成一堆竖线，Irene 明确否过）。每章小结直接用一句正文。

Markdown 骨架：

```markdown
# 《[主题]学习笔记》

### [一句话副标题]

> 学习包难度 [入门级 / 中级 / 进阶级] ｜ 面向 [程度] ｜ [基础] ｜ 生成日期 YYYY-MM-DD
> 这份是「把知识讲透」的正文。怎么走看 00，资料链接和术语看 01，视频看 02。

1. ……
```
